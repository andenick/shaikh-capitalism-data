## 1. Definition

**S1504** is the annual U.S. series for two interrelated variables, 1948-2010:
1. **gGDP** = growth rate of nominal GDP = `ln(GDP_t / GDP_{t-1})` from BEA NIPA T1.1.5 line 1
2. **pp** = relative new purchasing power = `(CR + CA) / GDP`, where `CR` = Total Domestic Credit (modern: DCS Domestic Claims) and `CA` = Current Account Balance from BEA NIPA T4.1.

Shaikh (2016) Figure 15.9 plots gGDP vs pp to illustrate the "new purchasing power" thesis that nominal demand growth equals real growth plus net new credit + net foreign demand.

## 2. Why it matters

Chapter 15's monetary theory of inflation rests on identifying new purchasing power with the sum of credit creation and net foreign inflows. S1504 operationalizes that identity for the postwar US and shows it tracks nominal GDP growth closely.

## 4. Construction

**Composite + formula**. The construction recipe per Appendix 15.1 p.895-896:

```
gGDP_t  = ln( GDP_t / GDP_{t-1} )                              (Nominal GDP growth)
dCR_t   = CR_t - CR_{t-1}                                      (New credit creation)
PP_t    = dCR_t + CA_t                                         (New purchasing power)
pp_t    = PP_t / GDP_t                                          (Relative new purchasing power)
```

The book values are read directly from `Appendix15_USInflation.xlsx` columns `gGDP`, `pp`, `gCR`, etc. for validation. The extension fetches modern equivalents via:
- BEA NIPA API (if `BEA_API_KEY` set) for T1.1.5 line 1
- IMF MFS_DC `DCORP_N_DC` via `_imf_ifs_resolver.fetch_ifs_series('DCORP_N_DC', 'USA', start, end)` for CR
- BEA NIPA API for T4.1 line 29 for CA

## 7. Caveats

1. **IMF IFS code remap** (NOT a proxy): legacy line numbers 31 + (78-88) + 79 + 81 retired post-2009; modern SDMX 3.0 publishes the same Domestic Claims aggregate under `DCORP_N_DC` in dataflow `MFS_DC`. Recipe in `_imf_ifs_resolver.LINE_TO_MODERN`. Code-remap, not concept substitution; registry records `code_remap: true`.
2. **Pre-2001 IMF IFS coverage gap**: IMF SDMX 3.0 API only publishes USA MFS_DC data from 2001Q1 onwards. The 1948-2000 segment lives only in IMF's archive bulk download or, as here, in Shaikh's hand-compiled values in the chopped table.
3. **BEA T1.16 line ordering**: not directly used here (used by S1505) but flagged for cross-series consistency.
4. **Cross-validation strategy**: if IMF resolver returns data for the 2001-2010 overlap, the validator compares modern `DCORP_N_DC` (USA) against Shaikh's hand-summed CR with +/- 2% tolerance per `_imf_ifs_resolver.validate_against_shaikh`.
