# S1604 — Extension Provenance Record

**Series**: S1604 — Net Average and Real Incremental Rates of Profit
**Phase**: 6 (Extension)
**Construction classification**: `composite`
**Extension method**: cascade — re-fetch r_corp/iroprcorp from S0608 RSCD chain, TB3MS from FRED, rebuild differences and re-run HP100 on extended series
**Authored**: 2026-05-18
**Author**: opus-subagent-ch16-fanout

---

## 1. Why this series is extendable

`time_series`. All upstream components are continuously published:
- `r_corp`, `iroprcorp` from S0608 (chapter-6 profit-rate chain) — extended via BEA NIPA/Fixed Assets through the RSCD chapter-6 chain
- `TB3MS` from FRED — direct continuation
- `ec_c` (counterfactual) from S1602 — re-derived via OLS forecast

## 2. Construction classification

`composite` with embedded `formula` derivation. Per the Anu rule: extension MUST re-fetch components and recompute the formulas, NOT lazy-splice a growth rate onto the difference series.

## 3. Method

```
INPUTS (each extension run)
  S0608 RSCD processed parquet -> r_corp_t (decimal, 1947-2024 if S0608 ext done)
  S0607 RSCD processed parquet -> iroprcorp_t (decimal)
  FRED TB3MS annual avg        -> i_3mo_t (percent -> /100 for decimal)
  S1602 _sidecars/S1602_counterfactual_fit.json + ec_c series

COMPUTE (post-extension, full 1947-2024 sample)
  S1604-A_t  = r_corp_t        - i_3mo_t/100.0     # net rate of profit
  S1604-B_t  = iroprcorp_t     - i_3mo_t/100.0     # net incremental, raw
  S1604-C    = hp_filter(S1604-B, lam=100)         # HP100 smoothed
  S1604-D_t  = S1604-C_{t-1}                       # HP100 lagged 1

COUNTERFACTUAL (S1604-E rcorpalt)
  Read ec_c_t, ec_t from S1602
  Compute ec_prime_t = ec_c_t / ec_t
  Compute Eccorp_alt_t = Eccorp_t * ec_prime_t      (from ProfitRates Appendix 16.2 mapping)
  Compute Pcorpalt_t   = Pcorp_t - (Eccorp_alt_t - Eccorp_t)   # higher labor cost reduces profit
  Compute rcorpalt_t   = Pcorpalt_t / K_t           # K_t from Appendix 6.8 chain
  S1604-E_t  = rcorpalt_t - i_3mo_t/100.0

SENSITIVITY (sidecar only, not canonical)
  S1604-C_sens = hp_filter(S1604-B, lam=6.25)      # Ravn-Uhlig
```

## 4. Component re-fetching

Every extension run re-pulls TB3MS from FRED (TTL 30 days), re-reads S0606-S0608 processed parquets, re-reads S1602 sidecar JSON. HP100 is recomputed on the full extended sample — the smoothed values shift by construction, and this is documented as expected behavior in the registry's `hp100_endpoint_fragility: true` flag.

## 5. Proxies

**None.** TB3MS is the same US Treasury 3-month auction-result rate Shaikh cites via ERP Table 73. Profit rate inputs trace to BEA NIPA / Fixed Assets via the chapter-6 chain. No concept substitutions.

## 6. Synthetic data

**None permitted.** If S0608 has not been extended past 2011, S1604 publishes the book period 1947-2011 only and the extension marker reads `extension_partial: pending_chapter_6_chain`. If TB3MS is missing for the splice year 2011, the loader walks back to 2010 / 2009; if no overlap, fails hard.

## 7. Failure modes & graceful degradation

| Failure | Detection | Action |
|---|---|---|
| FRED_API_KEY missing | startup | TB3MS extension skipped; book values used |
| S0608 not extended | parquet year-max check | Skip extension; mark `extension_status: blocked_by_S0608` |
| S1602 sidecar missing | file existence | `rcorpalt` not computed; mark `extension_partial: missing_S1602_counterfactual` |
| TB3MS unit ambiguity | sanity check abs(median)<1 | Convert percent->decimal if magnitude indicates percent |
| HP filter non-convergence | statsmodels exception | Emit raw S1604-B; HP variants marked unavailable; log to validator |

## 8. CD2 divergence

CD2's `S099` had subseries A (rcorp), B (iroprcorp), C (HP100 IROP), D (HP100 IROP lag(1)), E (rcorpalt). RSCD S1604 keeps the same 5-letter decomposition but:
- A is now r_corp - i (net), not gross r_corp (gross r_corp is S0608)
- The 5 CD2 subseries are net difference series, matching the Fig 16.8 plot directly

CD2's `S099` is the matched predecessor — values should align within 1% on the book period.

## 9. Splice diagnostics (V03)

- Net-difference identity: `S1604-A[t] - (r_corp[t] - TB3MS[t]/100)` should equal 0 to machine precision
- HP100 smoothed-vs-raw plot: per-year deviation tracked for visual inspection
- Endpoint sensitivity: the value of S1604-C[1947] computed with and without the latest year of data — reported as a diagnostic (typically 10-50 bp shift)
- rcorpalt cross-check: re-deriving `Pcorpalt` from S1602's book-period `ec_c` should reproduce the book's `Counterfactual Corp Profits` column within 0.5%
