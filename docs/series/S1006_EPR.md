# S1006 — Extension Provenance Record

**Series**: S1006 — Bond and Equity Returns
**Construction**: `composite` (three independent total-return series)
**Extension**: Ibbotson → Damodaran NYU substitution + LT Corp reconstruction

---

## 1. Why this series needs a substitute

Ibbotson SBBI is now a Morningstar commercial subscription product. Redistributing post-2010 Ibbotson values in an open replication would breach the license. Per Phase 4 decision 0005, the recommended open substitute is **Aswath Damodaran's NYU Stern dataset** "Historical Returns: Stocks, T.Bonds & T.Bills with Premiums" published at:

> https://pages.stern.nyu.edu/~adamodar/New_Home_Page/datafile/histretSP.html

(verified 200 OK 2026-05-18; updated annually by Damodaran)

## 2. Concept Match Justification — Damodaran vs Ibbotson

| Ibbotson SBBI series | Damodaran equivalent | Concept match |
|---|---|---|
| Large Company Stocks (S&P 500, total return incl. dividends) | "S&P 500 (includes dividends)" | EXACT — same index, same total-return convention |
| Long-Term Government Bonds (~20yr UST) | "US T. Bond" (10yr nominal, total return) | NEAR — Damodaran reports the 10yr T.Bond total return; Ibbotson's series is the ~20yr "Long Bond". Duration difference (~10yr vs ~20yr) introduces a small spread sensitivity, but both capture the "long-government nominal total return" concept. Flagged `proxy: false` because Damodaran openly documents this as the standard open replacement; the duration difference is a known small discrepancy. |
| Long-Term Corporate Bonds (~20yr investment-grade) | NOT PUBLISHED by Damodaran | NO direct match — see §3 |
| 3-month T-Bills | "3-month T.Bill" | EXACT |
| Inflation | "Inflation Rate" | EXACT (CPI-U) |

`proxy: true` is set only on the LT Corporate Bond extension (S1006-B-ext) — see §3.

## 3. LT Corporate Bond reconstruction (PROXY)

Damodaran does not publish a long-corporate total-return series. For the 2011+ extension of S1006-B (`rbcorplt`), we reconstruct an approximate total return from FRED `AAA` yield:

```
proxy_rbcorplt[y] ≈ AAA_yield[y]      # treat yield as running yield of duration-stable LT corp index
```

This is a deliberate concept *under*-statement (it omits the capital-gain component from yield changes) and is therefore flagged:

- `proxy: true` on S1006-B-ext
- `proxy_justification`: "Damodaran does not publish LT corporate total return; AAA yield reconstruction approximates the running-yield component of Ibbotson's ~20yr investment-grade corporate total return index. Omits ~5–7yr duration capital-gain contribution. Open-license alternative is the only viable substitute given that Bloomberg US Long Credit Index requires a paid subscription."

A future iteration could improve this with an explicit duration model (modified duration ~7yr × ΔAAA → capital gain), but Phase 4 ratified the simpler yield-only proxy with clear flagging as adequate for Phase 5 first pass.

## 4. Method

```
S1006-A (Equity)
  S1006-A_book[y]  = Ibbotson_xlsx['Large Company Stocks'][y]      1926-2010
  S1006-A_ext[y]   = Damodaran['rslarge'][y]                        2011-present
  (concatenate; no reindex needed — both are in percent total return)

S1006-B (LT Corp Bonds)   ⚠ proxy:true for extension
  S1006-B_book[y]  = Ibbotson_xlsx['Long-Term Corporate Bonds'][y]  1926-2010
  S1006-B_ext[y]   = FRED AAA yield[y]                              2011-present  (PROXY)

S1006-C (LT Gov Bonds)
  S1006-C_book[y]  = Ibbotson_xlsx['Long-Term Government Bonds'][y] 1926-2010
  S1006-C_ext[y]   = Damodaran['rbgovlt'][y]                        2011-present
```

## 5. Damodaran HTTP verification

Per Phase 4 adequacy: `HEAD https://pages.stern.nyu.edu/~adamodar/New_Home_Page/datafile/histretSP.html → 200 OK` (2026-05-18).

The loader calls `S00_apis.damodaran_histret()` which fetches the HTML, parses the table, normalizes percentage encoding (Damodaran reports decimal fractions; we multiply by 100 to match Ibbotson convention), and caches the result.

## 6. Synthetic data

**None.** If Damodaran fetch fails, the loader degrades to publishing the book period only.

## 7. Failure modes

| Failure | Action |
|---|---|
| Damodaran HTTP non-200 | Retry 3× w/ backoff; on failure → publish book period only, `extension_status: damodaran_unavailable` |
| Damodaran HTML schema change | Loader logs warning and falls back to book period |
| FRED AAA unavailable | LT Corp extension omitted; S1006-B publishes book only |
