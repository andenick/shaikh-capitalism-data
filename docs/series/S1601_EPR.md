**Series**: S1601 — US and UK Golden Waves
**Phase**: 6 (Extension)
**Construction classification**: `composite`
**Extension method**: re-fit cubic on extended sample 1786-2024 (component re-fetch + re-derive); baseline 1786-2010 published-trend variant retained as reference
**Authored**: 2026-05-18
**Author**: opus-subagent-ch16-fanout

---

## 1. Why this series is extendable

S1601 is `time_series`. The underlying WPI series (US: BLS WPS00000000 -> FRED `PPIACO`; UK: ONS K646 successor to historical WPI) and the gold price (London PM fix -> FRED `GOLDPMGBD228NLBM`) are all continuously published. The cubic-trend de-trender is a deterministic transform recomputed on every extension — no opaque smoothing carries forward.

## 2. Construction classification

`composite` — multiple input series (US WPI, UK WPI, gold price, optional historical splice chain) combined into a single output via a non-trivial transform (cubic-trend de-trending and 1930=100 rebase). The Anu rule on lazy splices requires re-fetching component data and recomputing the transform; this is exactly what the extension variant does.

## 3. Method

```
INPUTS
  Appendix5_DATALRprices.xlsx   = canonical book truth (1786-2010)
  FRED PPIACO                   = US WPI extension (1947-present, monthly -> annual avg)
  FRED GOLDPMGBD228NLBM         = LBMA Gold PM fix (1968-present, daily -> annual avg)
  ONS K646                      = UK PPI manufactured output (1989-present, monthly -> annual avg)
  BoE Millennium dataset        = UK historical WPI cross-check (research-use license)

VARIANT 1 — baseline (published trend)
  S1601-A_baseline[year] = USGoldWaveDetrended[year]    for year in [1786, 2010]
  S1601-B_baseline[year] = UKGoldWaveDetrended[year]    for year in [1786, 2010]

VARIANT 2 — extended (re-fit cubic on full sample)
  1. Construct USPPIGold_t = US_WPI_t / US_Gold_t      (1786-2024)
                via book table 1786-2010, FRED splice 2011-2024
  2. Construct UKPPIGold_t = UK_WPI_t / UK_Gold_t      (1786-2024)
                via book table 1786-2010, ONS + LBMA splice 2011-2024
  3. Fit OLS cubic on ln(USPPIGold) ~ year + year^2 + year^3
  4. Residual_t = ln(USPPIGold_t) - fitted_t
  5. Centered_t = Residual_t - Residual_{1930}
  6. S1601-A_extended[year] = 100 * exp(Centered_t)
  7. Repeat for UK -> S1601-B_extended[year]

SPLICE (extension years 2011-2024 of the extended variant)
  US WPI:   book WPS00000000 -> FRED PPIACO via growth-rate splice at 2010 overlap
  US Gold:  book Officer-Williamson -> FRED GOLDPMGBD228NLBM via overlap-anchor at 2010
  UK WPI:   book UKWPI -> ONS K646 via growth-rate splice at 1989 (historical UK WPI
            discontinued in 1989; ONS K646 is the successor PPI)
  UK Gold:  Jastram/Officer -> LBMA via overlap-anchor at 2010

OUTPUT
  Two variants per country, four total subseries:
    S1601-A_baseline, S1601-B_baseline (book-faithful 1786-2010 only)
    S1601-A_extended, S1601-B_extended (re-fit 1786-2024)
```

## 4. Component re-fetching

Per the composite-construction rule, the extended variant does **not** lazy-splice a growth rate onto the baseline residuals. It re-fetches the underlying WPI and gold price components, re-computes WPI/gold, re-fits the cubic on the longer sample, and re-derives the residual deviation. Endpoint sensitivity is the price of doing this faithfully.

## 5. Proxies

**None.** All components are primary published government / official sources:
- US WPI: BLS All Commodities, redistributed via FRED PPIACO under public-domain license
- UK WPI: ONS Open Government Licence v3.0 (K646 successor to historical UKWPI series)
- Gold: London PM fix / LBMA (publicly disseminated daily) via FRED redistribution

BoE Millennium dataset is used only as a cross-check on the UK historical chain, not as a spliced source — its values are documented in the registry for traceability but are not substituted into the active extension path.

## 6. Synthetic data

**None permitted.** If a FRED component returns NaN for a year, that NaN propagates through `WPI / gold` and the residual is NaN. No interpolation, no carry-forward.

## 7. Failure modes & graceful degradation

| Failure | Detection | Action |
|---|---|---|
| `FRED_API_KEY` not set | S00_config startup | Loader degrades to baseline variant only; extended variant marked `data_unavailable: api_key_missing` |
| FRED returns NaN at 2010 overlap | Processor checks | Walk back to 2008/2009 anchor; if none, FAIL with no extension |
| ONS K646 page unreachable | HTTP non-200 | UK extension marked unavailable; baseline UK remains valid 1786-2010 |
| BoE Millennium dataset out-of-date | Manual review | Cross-check only; not in critical path |
| Cubic fit non-convergence | `np.linalg.lstsq` warning | Document the warning in the diagnostic dict; do NOT silently substitute another smoother |

## 8. Splice diagnostics (reported by V03)

- Baseline variant: per-year MAE vs `USGoldWaveDetrended` / `UKGoldWaveDetrended` columns
- Extended variant: cubic coefficient estimates (4 per country) and goodness-of-fit R²
- Sample-year deviations at 1790, 1830, 1873, 1896, 1929, 1971, 2010 (book figure reference)
- Splice continuity diagnostics at 1989 (UK WPI -> ONS K646) and 2010 (book -> FRED)

## 9. CD2 divergence

CD2's `S093` decomposed into six subseries (US/UK detrended plus rescaled-at-1983-trough and rescaled-at-latest-peak variants). Our `S1601` keeps four canonical subseries (A-D) covering the two countries × two variants (baseline / extended). The two rescaled variants from CD2 are not emitted — they were aesthetic alignments for plotting, not new data. Documented here so a reviewer does not flag the missing CD2 subseries.
