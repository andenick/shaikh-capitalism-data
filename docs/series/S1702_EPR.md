## 1. Why this series cannot be extended

S1702 is a cross-sectional snapshot at a single tax year (2011). Per
anu-framework: "Extensions ONLY apply to time_series type." Cross-sectional
snapshots are not extended — each subsequent IRS SOI release (Publication
1304 Tax Year YYYY) is a separate series, not a continuation.

## 2. Future-tax-year reconstruction (not an extension)

A user wishing to update Figure 17.2 to a more recent tax year would:
1. Download IRS SOI Pub 1304 Table 1.4 for the desired year.
2. Apply the same bin-midpoint -> frequency -> cum-from-above pipeline.
3. Register it as a separate cross-sectional series.

This is NOT an extension of S1702 and would be a new RSCD series.

## 3. No-Proxy / No-Synthetic

The IRS is itself the canonical authoritative source. No substitution
required. The open-ended $10M+ bin has no midpoint in the workbook and is
not imputed — anti-synthetic.

## 4. CD2 divergence pre-disclosure

CD2's S103 reference_values were the Frequency column (FPR017_C3), not the
Cumulative-from-Above (FPR017_C5). Phase 4 corrected; RSCD v1 uses column F
and matches to <0.01%.
