# S1102 — Extension Provenance Record

**Phase**: 6 (Extension)
**Construction**: `formula` (eq. 11.16, REER PPI-basis)
**Extension method**: BIS PPI-based EER broad index, reindexed to Shaikh's 2002=100 anchor
**Status v1**: `not_attempted_v1` (deferred to Phase 9 enrichment)
**Proxy flag**: `true` (post-2009 extension uses BIS in lieu of Shaikh-method bespoke REER)

---

## 1. Why this series is extendable (and why with a proxy)

S1102 is a `formula` series. Per anu-framework's No-Lazy-Splices rule, the
extension must re-compute the formula with extended component data — it cannot
growth-rate splice the published rxr1 column. However, three of the four
required inputs are no longer maintained on Shaikh's original sources: BLS
International Labor Comparisons program was discontinued in 2013 (the program
that produced ULC Table 9 and exchange-rate Table 11). The only path forward
is to substitute a peer-reviewed published REER series — BIS's PPI-based EER
broad index — accepting a documented concept gap.

## 2. Concept Match Justification (BIS PPI EER vs Shaikh REER)

Both are **trade-weighted multilateral real exchange rates deflated by
producer prices**. Differences:

- BIS uses ~60 partner countries with rolling 3-year trade weights;
  Shaikh used a static 19-country basket with weights `w_j = (X_j+M_j)/sum`.
- BIS base is 2010=100; Shaikh used 2002=100. Reindexing is arithmetic via
  the 1994-2009 overlap window.
- Both deflate by producer prices (PPI).

These differences are **basket and weighting**, not concept. Both series
answer the same economic question: "How has the trade-weighted real
purchasing power of a country's currency moved relative to its trading
partners, deflated by producer prices?" Concept match: **acceptable**.

## 3. Method (planned, v1 = not attempted)

```
1. Fetch BIS PPI EER broad for US and JP, 1994-current, base 2010=100.
2. Reindex to 2002=100 using overlap with rxr1 column in 1994-2009 window:
       scale_c = mean(rxr1_c[1994:2009]) / mean(BIS_c[1994:2009])
       S1102-c[year] = BIS_c[year] * scale_c for year in [2010, current]
3. Splice: S1102[year<=2009] = rxr1[year]; S1102[year>=2010] = scaled BIS.
4. Flag proxy=True with this CMJ.
```

## 4. No-Proxy compliance

The substitution to BIS is a flagged proxy (`proxy: true`) with the CMJ above
recorded in registry. Required by Phase 4 Adequacy Q1 resolution.

## 5. No-Synthetic data

None. If BIS returns NaN for a year, it propagates.

## 6. Failure-mode table

| Failure | Detection | Action |
|---|---|---|
| BIS endpoint down | Retry 3x | Mark `extension_status: api_unavailable` |
| Pre-1994 BIS data missing | Loader check | OK — overlap window starts 1994 |
| Shaikh rxr1 NaN in 1994-2009 | Processor check | Use longest sub-window available |

## 7. CD2 divergence pre-disclosure

CD2's S061 used the rxr1 column directly without any post-2009 extension;
RSCD v1 reproduces CD2's book-period values exactly (MAE 0.0). The Phase 9
extension to BIS will introduce a documented proxy splice not present in CD2.
