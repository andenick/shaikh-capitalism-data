## 2. Footnote-24 initialization is PINNED

Per Phase 4 ratification + Phase 5 calibration finding: the initial value `prweq[1947] = preq[1947] + 6.75`, where the `adj = 6.75` parameter is taken directly from the workbook's published prweq column (verified by `prweq[1947] − preq[1947] = 6.75` exactly). Footnote 24 describes the *original* calibration logic (same-mean over `[1948, 2011]`); the book's published column hardcodes the resulting `adj`. We pin `adj = 6.75` exactly.

Extension MUST NOT re-anchor on `[1948, current_year]` — that would alter every historical `prweq` value and destroy the book replication.

Processor enforces this with a hard-coded constant `BOOK_ADJ = 6.75` (and `NORMALIZATION_INTERVAL = (1948, 2011)` retained for documentation).

## 3. Method (book-period, executed in Phase 5)

```
INPUTS
  preq[t]   = IntroPPrice col 25 ('Real Price.eq')          1947..2011
  dvr[t]    = IntroPPrice col 26 ('Real Div.eq')             1947..2011
  rI[t]     = S1007-B ('Adjusted Corp. Real IROP')           1948..2011  (NaN in 1947)

INITIALIZATION (footnote 24)
  Forward iteration assumes prweq[1947] = preq[1947] + adj.
  Find adj such that mean(prweq[1948..2011]) = mean(preq[1948..2011]).

  Closed-form: let f(adj) = mean over t in 1948..2011 of prweq_t(adj).
  prweq_t is linear in adj (sum of (1+rI_s) products), so adj is solvable by
  one bisection or linear root-find. We use a direct linear solve.

RECURSION
  for t in 1948..2011:
    prweq[t] = prweq[t-1] * (1 + rI[t]) - dvr[t]
```
