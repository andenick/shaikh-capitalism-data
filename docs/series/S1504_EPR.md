## 1. Why this series is extendable

S1504 is `time_series` + `composite`. All three component sources (BEA NIPA, BEA Current Account, IMF IFS Domestic Claims) are continuously published. The IMF IFS modernization to SDMX 3.0 / `MFS_DC` dataflow is a code remap, not a concept substitution.

## 2. Method

```
INPUTS (book period, authoritative)
  S1504_book = chopped table Appendix15_USInflation.xlsx columns gGDP, pp, gCR, GDP, CR, CA

INPUTS (extension period, re-fetched)
  BEA_GDP_T115 line 1 (Nominal GDP)
  BEA_CA_T41                 (Current Account Balance)
  IMF MFS_DC DCORP_N_DC USA  (Depository Corporations Survey Net Domestic Claims)

EXTENSION COMPUTATION (for year t in [2011, current_year])
  gGDP_t = ln(GDP_t / GDP_{t-1})
  dCR_t  = CR_t - CR_{t-1}
  pp_t   = (dCR_t + CA_t) / GDP_t
```

## 3. IMF IFS resolver integration

```python
from loaders._imf_ifs_resolver import resolve_ifs_line, fetch_ifs_series

# Map Shaikh's legacy line 32 (Total Domestic Claims) to modern code
modern_code = resolve_ifs_line(32)   # -> "DCORP_N_DC"

# Fetch modern DCS Net Domestic Claims for USA over the overlap+extension window
fr = fetch_ifs_series("DCORP_N_DC", "USA", 2001, 2025, dataflow="MFS_DC", unit="USD")
if fr.success:
    cr_modern = fr.values   # dict[year, float] in USD
```

## 4. No-proxy disclosure

**Code remap, not proxy.** IMF's own SRF continuity documentation states `DCORP_N_DC` IS the same Depository Corporations Survey Domestic Claims aggregate that Shaikh's 4-line composite (31 + (78-88) + 79 + 81) hand-sums. Registry records `code_remap: true, proxy: false`.

## 5. No-synthetic disclosure

**None permitted.** If IMF SDMX returns empty for pre-2001 USA, that segment is published as `data_unavailable_pre_2001` — never fabricated.

## 6. Failure-mode table

| Failure | Detection | Action |
|---|---|---|
| `BEA_API_KEY` missing | `S00_config` | Loader uses chopped values only; extension publishes only the columns recoverable from FRED proxies (GDP available as FRED `GDP`) |
| IMF SDMX egress blocked | `fetch_ifs_series.success = False` | CR extension marked `data_unavailable_imf_api_blocked`; book-period CR retained from chopped values |
| Pre-2001 IMF data missing | resolver returns no values for years < 2001 | Documented expected; book period uses chopped Shaikh values |
| BEA T4.1 line label changed | row-label validation in loader | Warn + skip extension; document in `extension_diagnostics` |

## 7. CD2 divergence pre-disclosure

CD2's S083/S084 used a similar but vintage-different BEA snapshot. gGDP expected to match within ~0.5%; pp may diverge by ~1-2% post-2000 due to BEA Current Account methodology revisions and IMF IFS rebases.

## 8. Validation against Shaikh via resolver

`_imf_ifs_resolver.validate_against_shaikh(modern_values, shaikh_values, tolerance_pct=2.0)` is called over the 2001-2010 overlap when the API succeeds; the validator's `cd2_comparison` field reports the result for audit.
