**Phase**: 6 (Extension)
**Construction classification**: `formula`
**Extension method**: re-fetch HCCSDODNS (book-spec numerator) and DPI (book-spec denominator) from FRED; recompute ratio with explicit unit conversion
**Authored**: 2026-05-18
**Author**: opus-subagent-ch16-fanout

---

## 1. Why this series is extendable

`time_series`. Both components are continuously published Federal Reserve / BEA series republished on FRED with public-domain license:
- `HCCSDODNS` — Households Debt Securities and Loans Liability, quarterly, 1947Q1-present
- `DPI` — Disposable Personal Income, BEA NIPA T2.1 line 27, monthly/quarterly, 1959-present (annual back to 1929)

Both have no methodological breaks across the splice year 2012.

## 2. Construction classification

`formula` — direct ratio of two underlying series. Per the Anu rule on lazy splices for derived quantities, extension MUST recompute the formula with extended component data, NOT lazy-splice a growth rate onto the book ratio. This is exactly what L01/P02_S1605 does.

## 3. Method

```
INPUTS (each extension run, TTL=30d)
  FRED HCCSDODNS  quarterly -> annual avg, millions USD
  FRED DPI        quarterly -> annual avg, billions USD SAAR

UNIT CONVERSION (CRITICAL — explicit in P02)
  # Dimensional analysis:
  #   HCCSDODNS units = millions USD
  #   DPI       units = billions USD SAAR
  #   ratio = HCCSDODNS / (DPI * 1000)
  #         = millions / (billions * 1000)
  #         = millions / millions
  #         = unitless
  # If we forgot the *1000, ratio would be 1000x too large.

COMPUTE
  ratio_extended_t = HCCSDODNS_t / (DPI_t * 1000.0)    for t in [1947, 2024]

SPLICE
  S1605-A_extended[t] = book HHDebt[t]         for t in [1975, 2012]    (billions USD)
  S1605-A_extended[t] = HCCSDODNS_t / 1000.0   for t in [2013, 2024]    (converted to billions)
  S1605-B_extended[t] = book HHDispPersInc[t]  for t in [1975, 2012]
  S1605-B_extended[t] = DPI_t                  for t in [2013, 2024]
  S1605-C_extended[t] = book HHDebtIncRatio[t] for t in [1975, 2012]
  S1605-C_extended[t] = HCCSDODNS_t / (DPI_t * 1000.0)  for t in [2013, 2024]

SANITY GATE
  assert 0.3 <= ratio_extended_t <= 1.5 for all t in [2013, 2024]
  # Households can be highly leveraged (peak ~1.35 of DPI in 2007), but
  # cannot be 1000x DPI. This catches the CMDEBT/DPI 1000x failure mode.
```

## 4. Component re-fetching

Every extension run re-pulls HCCSDODNS and DPI from FRED. The ratio is recomputed from extended component data; never lazy-spliced. The unit conversion is in the P02 code with an explicit dimensional comment (per anu-framework Unit Documentation rule).

## 5. Proxies

**None.**

- **HCCSDODNS** = Z.1 D.3 line 2 (households-only). Direct match to book spec.
- **DPI** = NIPA T2.1 line 27 (Disposable Personal Income). Direct match to book spec.

CD2's CMDEBT (households+nonprofits) and PI (pre-tax personal income) were wrong-concept proxies. Phase 4 RATIFIED the correction to HCCSDODNS/DPI. The substitution RESTORES book-spec fidelity rather than introducing a new proxy. `proxy: false` on both subseries with `proxy_justification: null`.

Documentation in registry:
```json
"proxy": false,
"proxy_justification": "RSCD restores Shaikh's stated book spec (Z.1 D.3 line 2 / NIPA T2.1 line 27). CD2's CMDEBT/PI substitution was the wrong-concept proxy; HCCSDODNS/DPI corrects it."
```

## 6. Synthetic data

**None permitted.** If FRED returns NaN for any year, that NaN propagates to S1605-C. No interpolation, no carry-forward. NaN years are reported in the validator's `gaps` field.

## 7. Failure modes & graceful degradation

| Failure | Detection | Action |
|---|---|---|
| FRED_API_KEY missing | startup | Loader skips extension; book period 1975-2012 published only |
| HCCSDODNS or DPI returns 5xx | retry 3× | Mark `extension_status: api_error`, publish book period only |
| Splice continuity violation at 2012 | merge check | If FRED ratio differs from book ratio at 2012 by >2%, log diagnostic; do NOT auto-substitute |
| Sanity-band violation (ratio outside [0.3, 1.5]) | post-compute assertion | RAISE AssertionError — this is the CMDEBT/DPI 1000x failure mode trap |
| FRED rebases series | TTL refresh | Next run re-fetches; ratio is base-invariant |

## 8. CD2 divergence

CD2's `S100` extension used CMDEBT/PI. Our canonical extension uses HCCSDODNS/DPI. Documented divergence:
- **At 2011 (last full CD2 year)**: CMDEBT/(PI*1000) vs HCCSDODNS/(DPI*1000) — measure the magnitude as evidence of the wrong-concept-proxy effect.
- Expected: 5-15% level difference (CMDEBT is ~6% larger than HCCSDODNS due to nonprofits; PI is ~12-14% larger than DPI due to pre-tax basis; net effect ~7-9% lower ratio on the CMDEBT/PI basis).

This divergence is the empirical confirmation that the CD2 proxies were materially wrong — and the validator emits the comparison as informational evidence in the V03 report.
