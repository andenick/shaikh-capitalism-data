# S1004 — Extension Provenance Record

**Series**: S1004 — Real Interest Rate HP-filtered
**Construction**: `formula` (derived)
**Extension**: recompute end-to-end on extended panel

---

## 1. Extendability

Yes — the inputs (`iblong` → FRED AAA, `USWPI` → FRED PPIACO reindexed) are extendable. The HP filter is recomputed on the entire extended panel.

## 2. Method

```
PANEL CONSTRUCTION
  i_book[t] = USLR['iblong'][t]  for 1857..2011
  i_ext[t]  = FRED AAA annual mean[t]  for 2012..present
  i = concat(i_book, i_ext)

  p_book[t] = USLR['USWPI'][t]  for 1857..2011
  p_ext[t]  = FRED PPIACO annual mean[t] * scale, scale = USWPI[2011]/PPIACO[2011]
  p = concat(p_book, p_ext)

REAL RATE
  iblongreal[t] = i[t] − ( p[t] / p[t-1] − 1 ) * 100

HP FILTER (two-sided, default padding)
  iblongrealHP3[t]    = HP(iblongreal, λ=3)        # book replication
  iblongrealHP6.25[t] = HP(iblongreal, λ=6.25)    # Ravn-Uhlig sensitivity
```

## 3. No lazy splice

The HP smoothed series is NEVER spliced. Adding a new year re-runs the filter end-to-end so the smoothed trend is internally consistent across the entire panel.

## 4. Endpoint problem disclosure

Adding new years near the right edge alters the smoothed trend over the prior ~5–10 years. This is the well-known HP endpoint problem; users comparing v1 vs v2 outputs of S1004-B/C should expect small differences near the right edge.

## 5. Proxies

None — direct continuation of the same FRED series families.

## 6. Failure modes

If FRED extension is unavailable, S1004 publishes book period only with `extension_status: api_unavailable`.

## 7. CD2 divergence

Informational only; CD2's S054 likely used a single λ (=3) and a slightly different padding convention.
