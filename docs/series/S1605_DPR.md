## 1. Definition

**S1605** is the annual ratio of US household debt to disposable personal income, 1975-2024. **Figure 16.9** ("Household Debt-to-Income Ratio, United States, 1975-2011") tracks the dramatic post-1980s rise in household leverage that underwrote the 1990s-2000s consumption boom.

The series is the household-side analogue to the corporate net-profit-rate trajectory in S1604: as wage suppression squeezed real income (S1602), households substituted debt for income, producing the leverage build-up culminating in 2007.

## 2. Why it matters in Chapter 16

Shaikh (2016, p. 735) writes: "Households, whose real incomes had been squeezed by the slowdown in real wage growth, were offered ever-cheaper debt in order to maintain growth in consumer spending. In consequence... the household debt-to-income ratio grew dramatically in the 1980s, which added fuel to the boom." Fig 16.9 is the empirical exhibit. S1605 is the data backing for that exhibit.

## 4. Construction

S1605 is **formula** — a direct two-series ratio.

### CD2 wrong-concept-proxy correction (RATIFIED by Phase 4)

CD2's extension used the WRONG concepts on both numerator and denominator:
- **Numerator**: CD2 used `CMDEBT` (Households + Nonprofit Organizations Credit Market Instruments Liability) — INCLUDES nonprofit organizations. Book spec is Z.1 D.3 line 2: households-only.
- **Denominator**: CD2 used `PI` (Personal Income, broader pre-tax). Book spec is BEA NIPA T2.1 line 27: Disposable Personal Income (after-tax).

Both are wrong-concept proxies per the anu-framework No-Proxies rule. RSCD S1605 RESTORES book-spec fidelity by using:
- **Numerator**: FRED `HCCSDODNS` (Households-only Debt Securities and Loans Liability) — exact match to Z.1 D.3 line 2
- **Denominator**: FRED `DPI` (Disposable Personal Income) — exact match to NIPA T2.1 line 27

### Unit conversion (CRITICAL)

**HCCSDODNS** is published in **millions USD**. **DPI** is published in **billions USD SAAR**. The naive ratio `HCCSDODNS / DPI` would be 1000× too large — this is the textbook failure mode flagged in the anu-framework Unit Documentation rule (originally manifested as the CD2 `CMDEBT/DPI` bug).

**The canonical loader formula**, with explicit dimensional analysis:

```python
# DIMENSIONAL ANALYSIS — see P02_S1605_construct.py:
#   HCCSDODNS units = millions USD
#   DPI       units = billions USD SAAR
#   To get a unitless ratio:
#     ratio = HCCSDODNS [millions] / (DPI [billions] * 1000 [millions/billion])
#           = HCCSDODNS [millions] / DPI [millions]
#           = unitless
#   This avoids the well-known CMDEBT/DPI = ratio * 1000 failure mode.
ratio = hccsdodns / (dpi * 1000.0)
```

## 7. Caveats

1. **CMDEBT/DPI 1000× failure mode**. The textbook example of why the anu-framework Unit Documentation rule exists. The P02 loader has an inline dimensional-analysis comment AND a sanity assertion that `0.3 < ratio < 1.5` (households can be highly leveraged but not 1000× DPI). Any value outside this band raises `AssertionError` and fails the loader.
2. **Z.1 D.3 line renumbering post-2013**. The Federal Reserve renamed Flow of Funds -> Financial Accounts in 2013, and table D.3 was restructured. FRED `HCCSDODNS` preserves the concept regardless of D.3 line numbering. The loader uses FRED series identifiers, not Z.1 line numbers.
3. **HHDispPersInc cross-check**. The book `HHDispPersInc` (S1605-B) is in billions USD. FRED `DPI` matches this within revisions. CD2's PI substitution would diverge by ~20-25% on the level.
4. **No CMDEBT or PI substitution**. Both are explicitly rejected by Phase 4. The CD2 extension is not used.
5. **No proxies in canonical path**. HCCSDODNS is the book's exact concept; DPI is the book's exact concept. `proxy: false`.
